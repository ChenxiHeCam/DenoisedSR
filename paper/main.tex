\begin{abstract}
\noindent
Symbolic regression---the recovery of closed-form equations from data---is a
powerful route to interpretable physical laws, but its cost grows steeply with the
number of candidate variables and operators. In a typical measurement table only a
few columns and a few operators actually appear in the governing equation, yet a
solver must still pay for every irrelevant branch of the search tree. Here we show
that a learned, observation-conditioned \emph{front-end} that predicts which
variables are relevant---a \emph{support prior}---substantially improves and
accelerates symbolic regression without changing the backend solver. The front-end
couples a graph-attention network over a variable co-occurrence graph with a
statistical-feature model, and recovers the true variables with recall~$1.000$ and
near-perfect per-task recall across three physics suites, so it essentially never
discards a real variable. Supplied to the evolutionary backend PySR under a fixed
time budget, the prior doubles exact-law recovery (from $30\%$ to $60\%$), reduces
the search space by $86\%$, and accelerates time-to-solution by $3$--$6\times$ on
typical formulas; on black-box tabular benchmarks it raises held-out predictive
accuracy while a same-size random-support control degrades it, isolating learned
guidance from dimensionality reduction. The approach is solver-agnostic, adds only
$\sim$40~ms per task, and keeps final law selection with a verifiable symbolic
backend.
\end{abstract}

\section{Introduction}
The automated discovery of governing equations from observation is a long-standing
goal of the physical sciences\cite{schmidt2009distilling,udrescu2020aifeynman,%
makke2024interpretable}. Symbolic regression (SR) addresses it directly: rather
than fitting an opaque function, SR searches the space of mathematical expressions
for a compact formula that both fits the data and remains
human-interpretable\cite{koza1992genetic,cranmer2023pysr}. The resulting equation
can be dimensionally checked, extrapolated and falsified, properties that black-box
predictors lack. This interpretability has made SR central to data-driven physics,
from rediscovering conservation laws to proposing new constitutive
relations\cite{brunton2016sindy,udrescu2020aifeynman,tenachi2023physo}.

The central obstacle is combinatorial. An SR backend searches over expression trees
whose leaves are drawn from the input variables and whose internal nodes are drawn
from an operator library; the number of candidate trees grows exponentially with
the number of admissible leaves and operators and with tree
depth\cite{lafuente2024srbench,petersen2021deep}. In physics problems this cost is
largely wasted: a governing law typically involves only a handful of the measured
variables and a small subset of the operator library, yet a solver run over all
columns and operators still explores an enormous number of branches that cannot
appear in the answer. Existing systems attack this with dimensional analysis,
problem decomposition, sparsity priors or efficient
enumeration\cite{brunton2016sindy,rudy2017pde,udrescu2020aifeynman,ruan2026pse},
but the raw width of the search space remains the dominant cost under any fixed
compute budget.

We take a complementary route: instead of designing a faster search, we
\emph{narrow the space the search must explore}. We frame search-space reduction as
a standalone, reusable front-end problem (Fig.~\ref{fig:concept}). Given a small
observation set $\mathcal{D}=\{(\mathbf{x}_i,y_i)\}_{i=1}^{q}$ with $q$ as small as
$50$--$100$, a learned model predicts which input columns (and, as an auxiliary,
which operators) are likely to participate in the governing law, and passes this
\emph{support prior} to a conventional SR backend. The backend then performs the
final symbolic search and model selection over the reduced space, so the output
remains a verifiable equation rather than a neural approximation.

This division of labour is deliberate and, we argue, the correct one given the
current state of neural SR. Models that decode the full equation directly from data
have advanced rapidly\cite{biggio2021nesymres,kamienny2022endtoend,landajuela2022dso,%
shojaee2024tpsr,kamienny2023deep}, but they remain unreliable at \emph{assignment}:
they often recover the right variables and operators yet place them in the wrong
terms, so exact symbolic equivalence stays low. Predicting \emph{support}, by
contrast, is a far easier and more robust target, and even a perfect-recall variable
filter hands a search backend an exponential head start while never making the true
law unreachable. Our contributions are: (i) we cast variable- and operator-support
prediction as a reusable, backend-agnostic SR front-end; (ii) we present a
graph-attention support predictor, ensembled with a statistical-feature model, that
attains recall~$1.000$ across three physics suites; and (iii) we quantify the
downstream effect on a fixed-budget backend---doubled exact recovery, $86\%$
search-space reduction and $3$--$6\times$ speedups---against a same-size
random-support control that isolates learned guidance from dimensionality reduction.

\section{Related work}
\noindent\textbf{Symbolic regression backends.} Genetic-programming search over
expression trees remains the workhorse of SR\cite{koza1992genetic,schmidt2009distilling},
with PySR providing a fast, configurable modern implementation whose variable and
operator sets can be restricted externally\cite{cranmer2023pysr}. Physics-tailored
systems add inductive structure: AI-Feynman exploits dimensional analysis and
recursive decomposition\cite{udrescu2020aifeynman}, SINDy imposes sparsity over a
library of candidate terms\cite{brunton2016sindy,rudy2017pde}, dimensional-constraint
search restricts units\cite{tenachi2023physo}, and parallel symbolic enumeration
shares sub-expressions for efficiency\cite{ruan2026pse}. Our front-end is
complementary to all of these: it emits a reduced variable/operator set that any
such backend can consume.

\noindent\textbf{Neural equation models.} Transformer and reinforcement-learning
approaches learn distributions over expressions and decode them from numerical
input\cite{biggio2021nesymres,kamienny2022endtoend,landajuela2022dso,petersen2021deep,%
dascoli2022deep}. Hybrid planners couple a neural proposal with tree
search\cite{shojaee2024tpsr,kamienny2023deep}, multimodal pre-training aligns
equation tokens with sampled numeric behaviour\cite{meidani2024snip,li2024mmsr},
and e-graph and diffusion methods offer alternative ways to represent or generate
equivalent symbolic forms\cite{zheng2025egen,bastiani2025diffusion}; generative
latent-space search has also been proposed\cite{li2026gensr}. These methods motivate
our front-end framing: the shared, reliable signal they extract is \emph{which
ingredients matter}, even when full end-to-end decoding remains brittle.

\noindent\textbf{Physics-aware neural modelling.} A parallel line builds physical
structure into neural models as an inductive bias---Hamiltonian and Lagrangian
networks\cite{greydanus2019hamiltonian,cranmer2020lagrangian}, neural differential
equations\cite{chen2018neuralode}, symbolic distillation of trained
networks\cite{cranmer2020discovering}, and concept-driven or abductive law-discovery
systems\cite{fang2025ainewton,srivastava2025ainoether,ying2025phye2e}. These support
our emphasis on keeping the final expression symbolic and verifiable while using the
neural component only to narrow the search.

\noindent\textbf{Representations, benchmarks and generalisation.} Because the
front-end consumes \emph{sets} of observations, permutation-invariant and
set-attention architectures are natural building
blocks\cite{zaheer2017deepsets,lee2019settransformer}; the contrastive and
latent components of representation learning that prevent collapse or encode
hierarchy are also relevant\cite{bardes2022vicreg,nickel2017poincare,sinha2024hyperbolic}.
We evaluate with the SRBench protocol\cite{lafuente2024srbench} and PMLB
datasets\cite{olson2017pmlb}, and connect to recent studies of pre-training
generalisation, LLM-assisted discovery and scientific latent
spaces\cite{voigt2024ood,shojaee2025llmsr,hallin2024universal,rogers2025whatwedontc}.
Our reported comparison is deliberately controlled: a learned support prior improves
a fixed backend under matched full-search and same-size random-support baselines.

\section{Results}

\subsection*{The front-end recovers every true physical variable}
We first evaluate the support prior intrinsically, without any backend. For each
task the front-end receives $q=100$ noisy observations whose columns include the
genuine variables together with random ``distractor'' columns, and must predict the
true variable subset. Across three suites of physical laws---AI-Feynman
(118~equations), a curated real-law benchmark (real591, 91~tasks) and an extended
physics-law set (1085~equations)---the predictor attains recall~$1.000$ (or $0.999$)
with perfect recall on essentially every task (Fig.~\ref{fig:support}). In other
words it almost never drops a variable that the law uses; its only error mode is
occasionally retaining a few extra noise columns, which merely costs the backend a
little additional search rather than making the law unreachable. Precision ranges
from $0.85$ to $1.00$, reflecting an intentional high-recall operating point:
removing a true variable is fatal to recovery, whereas keeping a spurious one is
recoverable downstream. For reference, a naive statistical baseline trained on few
records with name-matched labels reaches only recall~$0.48$, confirming that the
graph-attention model contributes the decisive signal.

\subsection*{A perfect-recall prior strengthens a fixed-budget solver}
We next insert the front-end before PySR\cite{cranmer2023pysr} and compare three
conditions at a fixed $10$~s per-task budget on a $30$-equation Feynman subset with
$\sim$20 distractor columns: (A) \emph{full PySR} over all columns and the full
$17$-operator library; (B) \emph{$+$variable prior}, restricting the columns to the
predicted support; and (C) \emph{$+$operator prior}, additionally restricting the
operator library. Restricting variables raises exact-law recovery (held-out
$R^2\ge0.999$) from $9/30$ ($30\%$) to $18/30$ ($60\%$)---a $+30$ percentage-point
gain---while lifting mean held-out $R^2$ from $0.919$ to $0.967$, reducing the
average column count from $23.2$ to $3.2$ (an $86\%$ reduction) and halving the
effective search vocabulary from $40$ to $20$ (Fig.~\ref{fig:pysr}). The variable
prior thus converts a large fraction of budget-limited failures into exact
recoveries simply by removing branches that could never appear in the law.

\subsection*{The gain is search guidance, not dimensionality reduction}
A natural concern is that any reduction in column count would help. We control for
this with a \emph{same-size random-support} baseline that retains the same number of
columns but selects them without using the observations. On $24$ PMLB/SRBench
black-box tabular tasks (three seeds, $q=100$, $5$~s budget), the learned support
raises mean held-out $R^2$ from $0.347$ to $0.495$ with $10$ distractor variables
and from $0.315$ to $0.488$ with $20$ distractors, using only $31\%$ and $18\%$ of
the input columns respectively; the same-size random support instead \emph{degrades}
performance to $0.220$ and $-0.028$ (Fig.~\ref{fig:pmlb}). At matched column count
the random control is far worse than full search, whereas the learned prior is
better---direct evidence that the front-end supplies observation-conditioned
information about \emph{which} columns matter, not merely \emph{how many}.

\subsection*{Pruning the search space accelerates time-to-solution}
Because variable filtering shrinks the terminal set of the expression tree from
$\sim$24 to $\sim$4 symbols---an exponential reduction in branching---it also
shortens the time to reach an exact fit. Measuring time-to-solution with early
stopping on exact recovery, the front-end yields $3$--$6\times$ speedups on typical
physics formulas (Fig.~\ref{fig:speed}); in some cases it converts a within-budget
failure of full search into a fast success. The exception is deep-tree formulas
whose bottleneck is expression \emph{depth} rather than variable count, where pruning
the variable set gives little benefit. Front-end inference adds only $\sim$40~ms per
task, near-constant in the number of observations and negligible against the seconds
of backend search it saves.

\subsection*{Operator priors are net-neutral but a useful safeguard}
Restricting the operator library in addition to variables (condition~C) does not
improve mean exact recovery (it is slightly lower, $14/30$), because operators are
intrinsically harder to infer from small, noisy domains and a mis-removed operator
is fatal. We therefore report operator prediction honestly as an ablation rather
than a primary result. It nonetheless has a clear conditional value: when
full-operator search returns a \emph{pathological} high-$R^2$ fit built from exotic
operators, restricting the library to the predicted operators reliably recovers a
sane solution (Fig.~\ref{fig:op}; e.g.\ held-out $R^2$ improving from $0.19$ to
$0.92$). The triggering signatures---an exotic low-probability operator, anomalous
complexity, or $R^2$ stuck just below~$1$---are visible at inference without the
ground truth, so an adaptive ``re-run with operator restriction when the fit looks
pathological'' policy is deployable in practice.

\section{Discussion}
Our results support a simple thesis: for symbolic regression of physical laws, a
learned \emph{support prior} is a high-leverage, low-risk intervention. Because a
perfect-recall variable filter never makes the true law unreachable, it can be
applied unconditionally; because it removes an exponential number of irrelevant
branches, it materially improves both the success rate and the speed of a
fixed-budget backend. The front-end is solver-agnostic---its output is a reduced
variable/operator set that any tree-search, enumeration or generative
backend\cite{cranmer2023pysr,ruan2026pse,li2026gensr} can consume---and cheap enough
($\sim$40~ms) to prepend to every run. The same-size random-support control shows the
benefit is genuine search guidance rather than mere dimensionality reduction, and
the black-box results indicate that the prior helps even when no symbolic ground
truth exists, by steering a fixed-budget solver toward the informative columns.

The honest limitation, shared with direct neural SR\cite{kamienny2022endtoend,%
landajuela2022dso}, is that the neural component cannot yet \emph{assign} variables
and operators to the correct positions in the expression reliably; this is precisely
why we leave final term assignment to a symbolic, verifiable backend rather than
emitting the equation end-to-end. Closing this assignment gap---through richer
compositional latents, explicit constant/exponent recovery, and contrastive
objectives that keep the observation and formula representations
aligned\cite{bardes2022vicreg,meidani2024snip}---is, in our view, the central open
problem in moving from a front-end prior toward reliable end-to-end neural
discovery. Even short of that goal, the present front-end is immediately useful: it
makes existing physical-law solvers both more successful and several times faster,
with a provably safe failure mode, and it is compatible with the dimensional and
physics-aware priors that already strengthen scientific
SR\cite{udrescu2020aifeynman,tenachi2023physo,cranmer2020discovering}.

\section{Methods}

\noindent\textbf{Problem setup.} Each task provides
$\mathcal{D}=\{(\mathbf{x}_i,y_i)\}_{i=1}^q$ with $\mathbf{x}_i\in\mathbb{R}^{m}$,
where only an unknown subset $S^\star\subseteq\{1,\dots,m\}$ of columns appears in
the target law $f^\star$. The front-end predicts $\hat S\approx S^\star$ (and an
optional operator set $\hat O$) and passes it to an SR backend run under a fixed
time budget. We deliberately pad tasks with random distractor columns to emulate the
realistic regime in which a measurement table contains many irrelevant channels.

\noindent\textbf{Front-end architecture.} The variable-support predictor is an
ensemble of two complementary models. (1)~A graph-attention
network\cite{lee2019settransformer,zaheer2017deepsets} operates on a graph whose
nodes are the input columns; node features combine per-column statistics (moments,
correlation with the target, monotonicity and dimensional descriptors) and edges
encode a variable co-occurrence prior learned from a knowledge graph of physical
laws together with observation-derived correlation. Attention over this graph lets
the model reason about which columns \emph{jointly} support a plausible law rather
than scoring columns in isolation. (2)~A random forest scores each column from the
same per-column statistical features. The final score is
$s(\text{col})=\mathrm{RF}(\text{col})+\sigma(\mathrm{GAT}(\text{col}))$, and a
column is retained if $s\ge\tau$ with a low threshold ($\tau=0.10$) chosen to favour
recall. An analogous graph-attention model with graph-level pooling produces the
multi-label operator prediction used for the optional operator prior.

\noindent\textbf{Training data.} The predictors are trained only on internal
physical-law records---formula templates with sampled observation tables drawn from
a curated knowledge graph of physical laws---never on the external evaluation suites.
Training pools of $8$k--$40$k formulas were used, with observation tables pre-sampled
across variable ranges and a $10$--$30$ distractor curriculum so the front-end is
robust to the number of irrelevant columns at test time. An adversarial
(GAN--graph-attention) variant was explored but gave no consistent gain and is not
used.

\noindent\textbf{Backend integration and evaluation.} We use
PySR\cite{cranmer2023pysr} as the downstream backend because it exposes variable and
operator restrictions directly and is reproducible locally; the same prior interface
applies to enumeration\cite{ruan2026pse} and generative\cite{li2026gensr} backends.
All comparisons fix the backend budget across conditions, so the only changed
quantity is the search space exposed to the solver. \emph{Exact recovery} denotes
held-out $R^2\ge0.999$. Support quality is measured by precision and recall against
the known law variables; the same-size random-support control retains the same number
of columns selected without the front-end, isolating learned guidance from
dimensionality reduction. Time-to-solution is measured with early stopping on exact
recovery, and front-end overhead is measured separately. All reported numbers are
produced by the released evaluation scripts from the recorded result artifacts.

\noindent\textbf{Data availability.} The external benchmarks (AI-Feynman,
PMLB/SRBench) are public\cite{udrescu2020aifeynman,olson2017pmlb,lafuente2024srbench};
the physical-law training records are materialised from public formula and range
information. The recorded result artifacts that back every table and figure are
released with the code.

\noindent\textbf{Code availability.} Code for the front-end, training, evaluation
and figure generation, together with the result artifacts and the trained
graph-attention weights, is released at \url{https://github.com/<anonymized>}.
